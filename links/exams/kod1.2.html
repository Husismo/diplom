КОД 1.2 Верстка

Данный экзамен сложнее чем Код 1.1, поскольку в нем вам нужно не просто создать сайт на конструкторе, а написать его с нуля, используя HTML и CSS, проверять отладку кода, тестирование, уметь находить ошибки, и многое другое. С полным списком можно ознакомиться сайте.

Выбор редактора

Начать нужно с выбора текстового редактора, в котором вы и будете писать весь ваш проект. Вы можете писать как и в обычном блокноте, однако существуют специально созданные редакторы, чтобы ваша работа была в разы продуктивнее и быстрее. ниже представлен список наиболее популярных текстовых редакторов для написания кода.

  • Visual Studio Code
  • Atom
  • Sublime Text 3
  • WebStorm

Я рекомундую использовать Visual Studio Code, посколько это удобный, бесплатный IDE, в котором есть множество плагинов и расширений с помощью которых вы можете как стилизовать свой код, чтобы быстро разбираться в нем, так и писать код быстрее.

Валидность кода

Вам нужно будет уметь выявлят ошибки, однако, хоть код и выглядит с виду рабочим, в нем может присуствовать различные мелкие ошибки, которые не видны вооруженным взглядом. Это может быть, банально, не закрытя скобочка, или не поставленный знак ";". Существую различные валидаторы, которые и помогут выявить ошибки и обозначить вам где они находятся, например вот один из таких. Вставьте или загрузите в него ваш код, и он обозначит вам ваши ошибки.

Система контроля версий

Если ваш код больше чем обычный лендинг, или вы работаете в команде, вы просто обязаны использовать различные системы контроля версий. Они нужны для того, чтобы хранить информацию о всех изменениях в вашем коде, начиная с самой первой строчки, и кооперироваться с вашим напарниками. Вы можете создать резервную копию вашего проекта, и в случае допущения ошибки быстро откатиться к ней. Одним из таких является Git.

Базовый HTML

Это то, без чего вам не выполнить основные задания. Вам нужно знать HTML и CSS. Освоить их можно быстро, за пару недель, если заниматься по несколько часов в день, существует множество шпаргалок и курсов по ним, которые вам обязательно помогут. HTMl это язык гипертекстовой разметки, это НЕ язык программирвоания. Для написания своей первой Web страницы, вам нужно знать парочку тегов, однако чтобы сайт выглядел красиво, вы должны уметь использовать еще и CSS.

HTML состоит из тегов, грубо говоря это каркас сайта, весь контент и информация, которая на нем находится. Существует множество тегов, такие как "<a></a>, <p></p>, <h1></h1>, <br>".

Обратите внимание, что существуют как закрытые теги, так и теги которые не нужно закрывать. Чтобы написать ваш перый сайт, создайте текстовый файл формата .txt, переиминуйте его в формать .html, откройте его в вашем текстовом редакторе и напишите туда, например, <h1>Привет мир!</h1> Все теги заучивать не нужно, для созданой хорошей веб страницы будет достаточно знать базовые теги, ниже предоставлены источники с помощью которых вы можете более подробно изучить об HTML.

HTML Book - на данном сайте собраны все существующие теги, можно использовать как шпарагалку.


Базовый CSS

Как и говорилось выше, CSS нужен для стилизации вашей Web страницы, без CSS ваша страница будет выглядеть скучно, старомодно и просто некрасиво. В CSS осваивается не очень долго, существуют такие же подсказки и шпаргалки. Благодаря CSS ваша страница будет выглядеть живой, можно будет добавить простенькую анимацию, например при наведении, как на кнопке ниже.

Наведите сюда

Обычно, код с CSS создается в отдельном файле с расширением .css и подключается через HTML. Данный файл содержит все стил ко всем объектам на странице. Обращение к объектам происходит через классы. В HTML вы должны задать атрибут Class="name", а после чего обратиться к нему в ксс через .name{свойсвтва}. Благодря css вы можете менять размеры картинок, задний фон, да и вообще абсолюнто все строится на css.
